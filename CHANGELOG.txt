================================================================================
  [v1.1.1]  Hotfix
  Release Date : March 26, 2026
  Status       : CURRENT — Production
================================================================================

  ── BUG FIXES ────────────────────────────────────────────────────────────────

  [FIXED] AttributeError: 'CalcApp' object has no attribute '_step_resultado'
      Root cause: when the chart helper block was injected during v1.1 dev,
      the "def _step_resultado(self):" declaration line was accidentally
      dropped. The method body was intact but the def line was missing,
      so Python never registered the method on the class. App crashed
      immediately after splash on every launch.
      Fix: restored "def _step_resultado(self):" above the method body.

  [FIXED] Language selection popup not centered on screen
      Root cause: winfo_screenwidth() / winfo_screenheight() were called
      before root.update_idletasks(). On Windows, Tk's display connection
      isn't fully initialized until a layout pass runs — winfo_screen*
      returned 1 as a fallback, placing the window at ~+239+154 instead
      of true screen center.
      Fix: moved root.update_idletasks() to run before the screen size
      query and geometry call. deiconify() moved earlier so the window
      is visible when idletasks flushes.
