================================================================================
  [v1.0 RC]  Release Candidate
  Release Date : March 15, 2026
  Status       : CURRENT — Production Ready
================================================================================

  This release incorporates all work from the cancelled v0.3 Beta cycle plus
  a full UI rewrite targeting the Windows 11 Fluent Design language. The app
  is now stable on Python 3.10 through 3.14 on Windows, macOS, and Linux.

  ── NEW COUNTRY ──────────────────────────────────────────────────────────────

  [+] Mexico / México 🇲🇽 — MXN currency support
      • IMSS — Enfermedad y Maternidad  ~2.50%
      • IMSS — Invalidez y Vida         ~0.63%
      • IMSS — Retiro (AFORE)           ~1.13%
      • IMSS — Guarderías y Prestaciones~1.00%
      • ISR   — 2024 progressive monthly table, 10 brackets (1.92% → 35%)
      • Amounts auto-formatted as MXN with thousand separators

  [-] USA / United States removed from country list
      • Previously included as a stub with incomplete data
      • Will return in v1.1 with complete Federal + FICA + State calculations

  ── UI ARCHITECTURE — COMPLETE REWRITE ───────────────────────────────────────

  [+] New dual-navigation system:
      • SideStepBar — vertical sidebar (Win11 Navigation Pane style)
        - Active step highlighted with left accent bar + blue background
        - Click any previously visited step to jump back directly
        - PIL-drawn circle icons per step (active=colored, inactive=gray)
        - Step name label below each icon
      • TopStepBar — horizontal breadcrumb pip row above content
        - 8 circle pips connected by thin lines
        - Colors: pending=gray, active=blue, done=green
        - Connecting lines turn green as steps are completed

  [+] FluentEntry — redesigned input field
      • 1px border frame with color-state management
      • Animated 2px bottom accent bar (Win11 text field focus indicator)
        - Idle:    gray border, gray bottom bar
        - Focused: blue border, blue bottom bar (#0067c0 / #60cdff dark)
      • Prefix label (currency symbol) on left side
      • Optional tooltip text shown on hover

  [+] FluentButton — redesigned button widget
      • Three variants: primary (blue), ghost (neutral), danger (red)
      • Press state: slightly darkened background on click
      • Hover state: accent-highlight background
      • Release state: returns to hover color
      • Optional PIL icon compound (left-aligned)

  [+] fluent_card() — redesigned card container
      • White (#ffffff) surface with 1px border (#e0e0e0)
      • 2px bottom shadow strip offset for depth without blur
      • Wrapper frame provides the shadow color strip

  [+] section_title() — redesigned section header
      • Small colored pill badge (PIL-drawn rounded rectangle) on left
      • Bold section title text
      • 1px accent-colored separator line underneath

  ── PIL ICON SYSTEM ───────────────────────────────────────────────────────────

  [+] _draw_circle_icon(size, bg, fg, symbol, scale)
      • Renders at 2× resolution then downsamples with LANCZOS anti-aliasing
      • Result: crisp, smooth icons at any DPI without image files
      • Font fallback chain: Segoe UI Bold → Arial Bold → DejaVu Sans Bold → default

  [+] _draw_pill_icon(w, h, bg, fg, symbol, radius, scale)
      • Rounded rectangle variant for section header badges
      • Same 2× supersample + LANCZOS pipeline

  [+] _icon_cache — all generated icons cached by key string
      • Prevents re-generation on theme switches and step changes
      • Cache is invalidated implicitly when theme colors change

  [+] make_step_icon(num, state, size) — step pip icons
      • state "pending": gray circle, gray number
      • state "active":  blue circle, white number
      • state "done":    green circle, white checkmark (✓)

  [+] make_sidebar_icon(step, active, size) — sidebar nav icons
      • Each step has a letter symbol (P, G, $, H, D, L, +, R)
      • Active: colored background matching step accent
      • Inactive: gray background, muted text

  NOTE: All symbols are ASCII/Latin characters (no emoji).
        PIL on Windows cannot reliably render Unicode color emoji
        from TrueType fallback fonts — this caused crashes in v0.3
        development. The ASCII approach is visually consistent and
        completely stable across all platforms.

  ── ANIMATION ENGINE ─────────────────────────────────────────────────────────

  [+] Tweener class — replaces all direct after()-based animation
      • run(start, end, ms, ease, on_update, on_done, fps)
      • Default: 60fps, 220ms duration
      • Easing functions:
        - Easing.out_cubic  — smooth deceleration (default for counts)
        - Easing.out_quad   — softer deceleration
        - Easing.in_out     — ease both ends (s-curve)
        - Easing.spring     — overshoot + settle (for UI pop effects)
      • animate_count(label, end_val, pais, ms) — drives result numbers
        from 0 to final value with out_cubic easing

  ── RESULTS PAGE — REDESIGNED ────────────────────────────────────────────────

  [+] Circular progress ring
      • Canvas arc showing savings % of total income
      • Color: green when positive, red when negative
      • Center text: percentage value
      • Drawn with progress_ring(canvas, cx, cy, r, pct, fg, bg, width)

  [+] Animated count-up on all three summary values
      • Total Income, Total Expenses, Net Savings
      • Count from 0 to final value with out_cubic easing on page load

  [+] Restructured layout: ring + stats side-by-side instead of list-only

  ── TOOLTIP SYSTEM ───────────────────────────────────────────────────────────

  [+] Tooltip class
      • Binds <Enter>/<Leave> to any widget
      • Shows a dark (#1f1f1f) popup 4px below the widget
      • 95% opacity, 9pt font, auto-positioned to widget coordinates
      • Automatically destroyed on mouse leave

  ── THEMES — FULL FLUENT SPEC ────────────────────────────────────────────────

  [+] THEME_LIGHT (default)
      • bg:          #f3f3f3  (Win11 app background)
      • surface:     #ffffff  (card surface)
      • accent:      #0067c0  (Win11 blue)
      • accent_soft: #cce4f7  (pill badge backgrounds)
      • title_bar:   #1f1f1f  (dark title bar always)
      • step_done:   #107c10  (Win11 green)
      • sidebar_bg:  #f0f0f0
      • sidebar_sel: #e0e0f0  (selected step)

      • bg:          #202020

      • accent:      #60cdff  (Win11 dark-mode blue)
      • title_bar:   #1a1a1a

      • sidebar_bg:  #1a1a1a
      • sidebar_sel: #2a2a40

  [+] _rebuild_theme() — swaps W11 dict in-place, triggers full redraw

  ── PYTHON 3.14 CRASH FIXES ──────────────────────────────────────────────────

  [FIXED] Crash on launch — dual tk.Tk() instantiation
      Root cause: Python 3.14 assigns _w = int (not ".") to any window
      created before root.update() is called. Child widgets fail with
      TypeError: unsupported operand type(s) for +: 'int' and 'str'

      Fix applied:
        root = tk.Tk()
        root.withdraw()
        root.update()     ← critical: flushes Tk internals
        root.deiconify()
        show_language_boot(root)   ← function, draws on root directly
        show_splash(root, cb)      ← function, draws canvas on root
        CalcApp(root)              ← tk.Frame, not tk.Tk

  [FIXED] Crash — tk.Label(font="")
      When PIL image is available, font= kwarg must be omitted entirely,
      not set to empty string. Tkinter on Windows treats "" as a font
      lookup and raises TclError: font "" doesn't exist.
      Fix: split into if/else Label construction.

  [FIXED] Crash — tk.Label(image=None)
      image= kwarg must only be passed when value is a real PhotoImage.
      Fix: build kw dict conditionally, unpack with **kw.

  [FIXED] Crash — STEP_ICONS used W11 dict at module load time
      W11 is populated by _rebuild_theme() which runs after import.
      Fix: replaced static STEP_ICONS list with _get_step_icon_colors(i)
      function that resolves colors at call time.

  [FIXED] Duplicate function definitions from refactoring
      section_title, note_box, tag_label, stat_row, w11_card, StepBar
      all had orphaned copies from the rewrite. Removed all stale copies.

  [v0.3 Beta]  CANCELLED — Never Released
  Target Date  : Never shipped
  Status       : CANCELLED — Merged into v1.0 RC
================================================================================

  v0.3 Beta development began after v0.2 Beta but was cancelled before any
  public release. All completed work was merged into and shipped as v1.0 RC.

  ── COMPLETED BEFORE CANCELLATION ────────────────────────────────────────────

  [+] Mexico 🇲🇽 tax calculator (calcular_impuestos_mexico)
      IMSS brackets + ISR 2024 monthly progressive table
      MXN currency formatting with thousand separators

  [+] USA 🇺🇸 tax calculator stub (calcular_impuestos_usa)
      Social Security 6.2% + Medicare 1.45% + simplified federal estimate
      Status: incomplete — not suitable for release. Removed in v1.0 RC.

  [+] Canvas-drawn country flags (no image files)
      draw_flag_panama, draw_flag_colombia, draw_flag_mexico, draw_flag_usa
      All drawn with tk.Canvas primitives (rectangles, ovals, polygons)
      Reused in the combobox flag preview and page title header

  [+] Win11 Light and Dark themes (initial implementation)
      THEME_LIGHT and THEME_DARK dicts, _rebuild_theme(), toggle_dark()
      Less complete than the v1.0 RC Fluent spec — no sidebar tokens,
      no progress_bg, no accent_soft

  [+] Animated splash screen (first version)
      Bouncing coin animation using canvas.move()
      Progress bar with eased fill
      Country flag strip drawn on canvas

  [+] Language boot screen (first version)
      Separate Toplevel window — this caused crashes on Python 3.14
      Replaced in v1.0 RC with a function that draws directly on root

  ── REASON FOR CANCELLATION ──────────────────────────────────────────────────

  Repeated crash bugs on Python 3.14 (Windows) made the build unsuitable
  for any public release. Rather than patch incrementally, the decision was
  made to freeze v0.3, rebuild the UI layer from scratch with correct
  single-root architecture, and ship the result directly as v1.0 RC.

      Step 5: Debts & Auto & Pets — loans, gas, maintenance, pet costs

      • Educational Insurance: 1.25% of gross salary

        25% on income above $50,000/year
      • Pay stub override: enter actual received salary to bypass estimate

  [+] Bilingual interface — English / Español

  [+] PDF export via reportlab
      • Full bilingual formatted report
      • Salary & tax breakdown table
      • Expense category totals
      • Goal analysis summary (savings needed vs. projected)

  [+] Animated splash screen
      • Bouncing coin animation

      • 8 steps shown as numbered circles
